MacHelper Pro 개인정보 처리방침

시행일: 2026년 6월 2일

MacHelper Pro는 미러링된 Mac 설정에서 내장 디스플레이를 제어하고, 선택적으로 Mac이 더 강한 저장된 Wi-Fi로 전환하도록 돕는 로컬 우선 macOS 메뉴 막대 유틸리티입니다.

이 번역은 편의를 위해 제공됩니다. 이 정책의 공식 버전은 영어 버전입니다.

앱 계정 없음 MacHelper Pro 로그인 또는 사용자 프로필이 없습니다.
분석 없음 이벤트 추적, 세션 추적, 광고 픽셀 또는 사용량 원격 측정이 없습니다.
Wi-Fi 비밀번호 없음 Wi-Fi 전환은 macOS 네트워크 세부 정보를 사용합니다. MacHelper Pro은 비밀번호를 읽지 않습니다.

  • MacHelper Pro 계정은 필요하지 않습니다.
  • 분석, 추적 SDK, 광고 SDK 또는 충돌 보고 SDK는 포함되지 않습니다.
  • Xintech LLC에는 백그라운드 원격 분석이 전송되지 않습니다.

로컬 데이터

MacHelper Pro은 소량의 로컬 기본 설정 데이터와 제한된 로컬 진단 로그를 Mac에 저장할 수 있습니다. 이는 출시 간에 앱이 일관되게 작동하도록 하고 버그 보고를 선택할 때 버그 보고 초안을 준비하는 데에만 사용됩니다.

현지 선호 사항에는 다음이 포함될 수 있습니다.

  • 자동 일시 중단이 활성화되었는지 또는 비활성화되었는지 여부입니다.
  • 시작 시 내장 미러링을 활성화해야 하는지 여부입니다.
  • MacHelper Pro 메뉴 작업에 대한 사용자 정의 키보드 단축키입니다.
  • Auto Wi-Fi Switch의 활성화 여부와 신호 강도 임계값입니다.
  • 미러링을 위해 기본 외부 디스플레이를 기억하는 데 사용되는 로컬 디스플레이 식별자입니다.
  • 버그 보고서 초안에 사용되는 MacHelper Pro 진단 메시지의 제한된 로그입니다.
  • 평가판 액세스를 관리하기 위해 평가판 시작 및 만료 표시가 로컬 및 Keychain에 저장됩니다.
  • 라이선스를 활성화하는 경우 라이선스 키와 Lemon Squeezy 라이선스 인스턴스 식별자가 Mac의 Keychain에 저장됩니다.
  • 오프라인 유예 동작에 사용되는 마지막 성공적인 유효성 검사 시간과 같은 비밀이 아닌 라이선스 타임스탬프입니다.

앱은 macOS 디스플레이 API를 사용하여 디스플레이 레이아웃을 검사하고, 미러링을 감지하고, 내장 디스플레이 밝기를 조정하고, 활성화할 때 "Dim Displays Until Key Press," 관리 Start at Login을 선택할 때 활성 디스플레이를 일시적으로 어둡게 하고, 필요할 때 고정된 안전 밝기 값을 복원합니다. 일시적인 밝기 조절의 경우 MacHelper Pro은 디스플레이를 복원할 수 있도록 키를 눌렀는지 여부만 확인합니다. 키 이름, 입력된 텍스트 또는 키보드 기록은 기록되지 않습니다. MacHelper Pro은 이전 밝기를 저장하지 않습니다.

Wi-Fi 상태 및 위치 권한

Auto Wi-Fi Switch은 선택 사항입니다. 활성화되면 MacHelper Pro은 공개 macOS CoreWLAN API를 사용하여 현재 Wi-Fi 이름, 액세스 포인트 주소, 신호 강도 및 대역을 읽고 더 나은 대안이 될 수 있는 저장된 자동 가입 네트워크를 검색합니다.

macOS에서는 앱이 Wi-Fi 네트워크 이름과 액세스 포인트 주소를 볼 수 있으려면 위치 권한이 필요할 수 있습니다. MacHelper Pro은 메뉴와 Auto Wi-Fi Switch에서 사용되는 Wi-Fi 세부 정보에 대해서만 해당 권한을 사용합니다. 앱은 위치 기록을 수집하지 않고, Wi-Fi 세부 정보를 Xintech LLC로 보내지 않으며, Wi-Fi 비밀번호를 읽거나 저장하거나 전송하지 않습니다.

네트워크 활동

MacHelper Pro의 일반 디스플레이 제어에는 네트워크 연결이 필요하지 않습니다. 자동 업데이트 확인이 활성화되거나 메뉴에서 "Check for Updates..."을 선택하면 앱이 공개 Sparkle 업데이트 피드에 접속할 수 있습니다.

업데이트 확인은 최신 버전이 있는지 확인하는 데에만 사용됩니다. 광고, 분석, 계정 추적 또는 사용 추적에는 사용되지 않습니다.

"Activate License..."을 선택하면 MacHelper Pro이(가) Lemon Squeezy에 연락하여 라이선스를 활성화, 확인 또는 비활성화합니다. 라이선스 요청은 Mac의 로컬 컴퓨터 이름을 기반으로 라이선스 키와 인스턴스 이름(예: "Sam's Mac의 MacHelper Pro")을 보내므로 나중에 해당 Mac을 인식하고 비활성화할 수 있습니다.

Lemon Squeezy은 라이선스 확인에 필요한 라이선스, 제품, 주문 및 고객 메타데이터를 반환할 수 있습니다. MacHelper Pro은 라이센스 키, 로컬 라이센스 인스턴스 식별자, 로컬 평가판 마커 및 로컬 유효성 검사 타임스탬프만 저장합니다. 라이센스 요청은 분석, 광고 또는 사용 추적에 사용되지 않습니다.

버그 보고서, 기능 요청 및 피드백

"Report a Bug...", "Suggest a Feature..." 및 "Send Feedback..." 메뉴 항목은 Xintech LLC로 주소가 지정된 기본 이메일 앱에서 초안을 엽니다. 앱은 자동으로 이메일을 보내지 않습니다. 초안을 보내기 전에 검토, 편집 또는 삭제할 수 있습니다.

초안에는 지원팀이 컨텍스트(앱 버전, macOS 버전, 프로세서 아키텍처, Auto Blackout 상태, 내장 미러링 상태 및 현재 앱 상태)를 이해할 수 있도록 간단한 진단 요약이 포함될 수 있습니다. 귀하의 이메일 주소는 귀하가 이메일을 보내는 경우에만 Xintech LLC에게 표시됩니다.

버그 보고서 초안에는 최근 MacHelper Pro 진단 로그의 제한된 발췌도 포함됩니다. 이러한 로그에는 로컬 디스플레이 상태, 앱 상태 세부정보, Wi-Fi 네트워크 이름, 액세스 포인트 주소, 신호 강도 및 Auto Wi-Fi Switch 결정이 포함될 수 있습니다. 기능 요청 및 일반 피드백 초안에는 로그가 포함되지 않습니다.

진단 및 로그

MacHelper Pro은 Apple의 로컬 통합 로깅 시스템에 진단 메시지를 기록하고 소규모의 제한된 앱-로컬 진단 로그를 유지합니다. 이러한 로그는 Mac에 로컬로 저장됩니다. "Report a Bug..."을 선택하면 앱 로컬 로그에서 제한된 MacHelper Pro만 발췌한 내용이 수집되어 이메일 초안에 삽입됩니다.

로그 발췌에서는 사용 가능한 경우 MacHelper Pro 로그의 마지막 20분을 사용합니다. 최근 항목이 너무 적으면 앱은 최대 24시간까지 되돌아보고 가장 최근 MacHelper Pro 항목을 포함할 수 있으며 최대 180줄, 16,000자로 제한됩니다. 보내기를 선택하지 않으면 아무 것도 전송되지 않으며, 로그 발췌문을 먼저 삭제할 수 있습니다.

사용자 선택

  • 피드백 초안을 보내지 않은 상태로 두거나 편집하거나 진단 세부 정보를 삭제할 수 있습니다.
  • 버그 보고서를 보내기 전에 진단 요약이나 로그 발췌문을 삭제할 수 있습니다.
  • Sparkle 업데이트 기본 설정에서 자동 업데이트 확인을 비활성화할 수 있습니다.
  • MacHelper Pro 메뉴에서 Auto Wi-Fi Switch을 켜거나 끌 수 있습니다.
  • macOS 시스템 설정에서 Wi-Fi 세부정보에 대한 위치 권한을 부여, 거부 또는 취소할 수 있습니다.
  • 평가판 기간 동안 라이선스를 활성화하지 않고도 Pro 디스플레이 컨트롤을 사용할 수 있습니다.
  • 라이센스 창에서 이 Mac을 비활성화할 수 있습니다.
  • 일반적인 macOS 환경 설정 관리 도구를 사용하여 로컬 앱 환경 설정을 제거할 수 있습니다.
